The Augmented Product Manager

Published on January 1, 2026 by Wayne Correa

Recovering from shoulder surgery with one arm in a sling, I was already feeling sidelined when a friend hit me with a concrete challenge: Can we visualize data from PeeringDB.com in a way that reveals how companies actually interconnect, without days of manual work?

What started as a small “what-if” experiment using AI to explore that question quickly evolved into cloudinfrastructure.org, a working solution that maps interconnection patterns across networks, facilities, and cloud on-ramps. This was not a thought exercise; it is a real example of using AI as a tool to move from constraint to a shipped solution in days, not months.

For Product Managers, especially in cloud networking, this story is a blueprint for turning vague, complex infrastructure questions into concrete tools that your customers can actually use.

PeeringDB: Powerful, but Hard to Interrogate

If you work in networking, PeeringDB is already familiar. It is the community-driven registry of networks, facilities, Internet Exchange Points (IXPs), and private interconnection points: a shared source of truth for how the internet physically fits together.

However, PeeringDB is optimized for record lookup, not for answering cross-cutting, strategic questions. It is a rich library with a limited card catalog: you can easily look up a single network or facility, but it is painful to ask questions like:

  • Across these two specific networks, where do they privately interconnect?
  • Which facilities emerge as critical hubs for private, network-to-network connectivity?
  • How does a particular provider’s private interconnect footprint compare across regions?

A friend framed the problem very specifically: “Which data centers are the most critical hubs for private interconnection between networks X and Y?”

Answering that with manual exports and spreadsheets would take days. With one arm immobilized, even that brute-force option was off the table, which turned into a forcing function to find a higher-leverage approach.

My Projects

Cloud Infrastructure Insights from PeeringDB

This project provides a tool to visualize and analyze interconnection data from PeeringDB, the source of truth for internet infrastructure. While PeeringDB is an excellent directory, it is not optimized for strategic analysis. This tool addresses that gap by answering complex questions about how networks, data centers, and cloud on-ramps connect globally.

It transforms raw, record-based data into an actionable map of the internet's physical topology. Users can explore which data centers are critical interconnection hubs, compare the private interconnect footprints of different providers, and identify strategic opportunities for network expansion. The project was born from a real-world need to move beyond manual, time-consuming data analysis and now serves as a public resource for infrastructure planners and network strategists.

  • Date: 2025-2026
  • Tools/Code: Python, AI-assisted data normalization, HTML, CSS, JavaScript
  • Website: cloudinfrastructure.org
